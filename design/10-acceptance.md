## 3. 游戏规则（功能）

### 3.1 棋盘与单位
- [ ] 棋盘尺寸由地图配置驱动；默认对战地图为 `frontier_30`（30 × 30）
- [ ] 双方各 1 个基地，位置对称
- [ ] 默认对战地图包含 6 个资源点、2 个前哨站与成组障碍格
- [ ] 双方各 3 个初始单位（1 战士 + 1 弓手 + 1 骑士）
- [ ] 招募池包含 8 个兵种：战士 / 弓手 / 法师 / 骑士 / 牧师 / 炮手 / 侦察兵 / 毒师
- [ ] 8 个兵种数值严格符合 `01-gameplay-rules.md` §3.1

### 3.2 移动
- [ ] 曼哈顿距离 ≤ 移动力
- [ ] 不能穿越敌方单位
- [ ] 可以穿越己方单位，但不能停在己方单位上
- [ ] 不能进入对方基地格
- [ ] 不能进入任何基地格（含己方）
- [ ] 可以进入资源点
- [ ] 移动后 `hasMoved = true`
- [ ] 单位每回合最多移动一次

### 3.3 攻击
- [ ] 射程按曼哈顿距离判定
- [ ] 弓手 [2,3]：距离 1 无法攻击
- [ ] 法师 [1,4]
- [ ] 战士 / 骑士 [1,1]
- [ ] 牧师普通攻击 [1,2]
- [ ] 毒师普通攻击 [1,2]
- [ ] 炮手普通攻击 [3,4]
- [ ] 攻击后 `hasActed = true`
- [ ] 单位每回合最多攻击一次
- [ ] 攻击后不能再移动
- [ ] 伤害 = `max(1, floor((ATK - DEF) × counterMul))`
- [ ] 克制 ×1.25，被克 ×0.8
- [ ] 克制链：战士→弓手→法师→骑士→战士
- [ ] 基地无克制加成
- [ ] 无反击
- [ ] 单位血量 ≤ 0 → 从场上移除
- [ ] 炮手可以指定空地中心点发动普通攻击
- [ ] 炮手 AOE 中心格全伤，四邻格溅射 0.5 倍，且不误伤友军

### 3.4 技能与特殊兵种
- [ ] `USE_ABILITY` 已实现并可由服务器校验
- [ ] 牧师可以治疗射程内受伤友军
- [ ] 牧师不能治疗自己、敌军、基地、满血单位、死亡单位
- [ ] 牧师可先移动再治疗
- [ ] 治疗后 `hasActed = true`
- [ ] 毒师普通攻击命中存活敌军后附加 1 层中毒，且不对基地施毒
- [ ] 毒爆可对施法距离 1–3 内任意中心格释放，允许空地与迷雾盲放
- [ ] 毒爆对中心及曼哈顿距离 ≤ 2 的敌军附加 2 层中毒，不影响友军与基地
- [ ] 中毒最多 5 层
- [ ] 中毒在受害方回合开始先造成等于层数的伤害，再衰减 1 层
- [ ] 中毒致死单位不会参与该回合后续收入 / 招募 / 维护费结算
- [ ] 毒爆使用后进入 2 个自己的回合冷却
- [ ] 炮手盲射空地时，服务器状态与客户端快照保持一致

### 3.5 经济
- [ ] 初始金币 5
- [ ] 基地每回合开始 +6
- [ ] 己方归属资源点每个 +3
- [ ] 场上单位在己方回合开始支付固定维护费
- [ ] 维护费数值符合：战士1 / 弓手1 / 法师2 / 骑士1 / 牧师1 / 炮手2 / 侦察兵1 / 毒师2
- [ ] 新招募单位在出生当回合立刻计入维护费
- [ ] 维护费不足时金币只扣到 0，不会进入负数
- [ ] 资源点归属：己方单位移动到资源点时 → 立即归属己方
- [ ] 资源点无人时保持上次归属
- [ ] 资源点被对方移动占领 → 归属对方
- [ ] 前哨站初始中立；占领前不产金、不招募、无光环
- [ ] 前哨站归属：任意单位移动到前哨站本格时 → 立即归属该方
- [ ] 己方已占领前哨站在己方回合开始提供 `+5` 金
- [ ] 前哨站本格与四邻格内的己方单位获得 `+2 DEF`；基地不吃加成；多个前哨站不叠加

### 3.6 招募
- [ ] 可对己方基地下招募单
- [ ] 可对己方已占领前哨站下招募单
- [ ] 金币不足拒绝
- [ ] 人口已满（≥ 16）拒绝
- [ ] 同一建筑本回合已下单拒绝（基地与每个前哨站各自独立）
- [ ] 中立 / 敌方前哨站下单拒绝
- [ ] 基地来源的 `spawnAt` 必须是基地邻格
- [ ] 前哨站来源的 `spawnAt` 必须是前哨站本格或邻格
- [ ] 金币立即扣除
- [ ] 下回合开始时生成单位
- [ ] 基地来源：生成位置被占 → 尝试其他邻格 → 全占则退钱
- [ ] 前哨站来源：前哨站本格优先 → 再尝试邻格 → 全占则退钱
- [ ] 新单位当回合 `spawnedThisTurn = true`，且可像普通单位一样行动

### 3.7 胜负
- [ ] 基地 HP ≤ 0 → 对方胜
- [ ] 投降 → 对方胜
- [ ] 备用时间耗尽 → 对方胜
- [ ] 不以"无单位+无金币"为胜负条件

### 3.8 时间
- [ ] 每回合 75 秒
- [ ] 每方备用 300 秒
- [ ] 回合倒计时归零 → 先扣备用时间，再继续
- [ ] 备用时间归零 → 判负

## 12. 最终演示脚本

作为验收的"最终 demo"，执行以下流程应该全部顺畅：

1. `docker compose up -d`
2. `pnpm install`
3. `pnpm --filter @chesspvp/server prisma migrate dev`
4. `pnpm dev`
5. 打开浏览器 A：`http://localhost:5173` → 游客登录 → "开始匹配"
6. 打开浏览器 B（隐身模式）：同样流程
7. 双方自动配对 → 进入战斗
8. 双方互相操作：移动、攻击、招募、占领资源点 / 前哨站
9. 一方摧毁对方基地 → 结算弹窗
10. 双方"返回大厅"
11. 在 server 日志里能看到结构化日志
12. 在 psql 里能查到一条 match 和对应 replay
13. 运行一个 Node 脚本：`node scripts/verify-replay.js <matchId>` 能从 DB 重建到最终状态并打印 "PASS"
